## Принципы

1. **Пользователь входит один раз** — сессия живёт долго, продлевается в фоне автоматически.
2. **Разрыв сети ≠ logout** — если сервер недоступен при старте, показываем офлайн-режим, не разлогиниваем.
3. **Минимум трения** — форма регистрации простая (4 поля), сразу пускаем в приложение без блокировки на верификации.
4. **Безопасность без CAPTCHA** — защищаем без раздражающих пользователей виджетов.

## Способы регистрации

### ✅ Реализовано: Email + Password (Фаза 1)

Основной способ. Четыре поля:

| Поле | Обязательно | Описание |
|---|---|---|
| `email` | ✅ | Скрыт от других, только для верификации и восстановления |
| `display_name` | ✅ | Отображаемое имя (2–50 символов), **не уникальное** |
| `password` | ✅ | Минимум 8 символов, должен содержать буквы и цифры |
| `passwordConfirm` | ✅ | Совпадение проверяется на клиенте через Zod |

Технически: `pb.collection('users').create(...)` + автоматический логин после регистрации.

PocketBase-хук `pb_hooks/main.pb.js` автоматически создаёт системную комнату «Избранное» после регистрации.

---

### ⬜ Запланировано: OAuth2 (Фаза 2)

PocketBase поддерживает OAuth2 «из коробки» через `authWithOAuth2()`.

**Кандидаты на подключение:**

| Провайдер | Ценность | Сложность |
|---|---|---|
| **Google** | Самый популярный, есть у всех | Низкая — только Client ID/Secret в PB admin |
| **GitHub** | Аудитория технарей | Низкая — то же самое |
| **Apple** | Обязателен для iOS App Store | Высокая — требует Developer Account |

**Поток OAuth2 в PocketBase:**

```
[Пользователь] → кнопка «Войти через Google»
    → pb.collection('users').authWithOAuth2({ provider: 'google' })
    → Открывается popup с Google OAuth
    → PocketBase получает токен, создаёт/обновляет пользователя
    → Возвращает стандартный AuthStore с JWT
    → Те же `pbUser` и `profile` в Zustand store
```

> **Важно архитектурно:** OAuth2 возвращает тот же тип `AuthUser`, что и логин по паролю. Вся остальная система (E2E ключи, хуки, стор) работает без изменений.

**Особенность с E2E шифрованием:** При OAuth-входе нет пароля → нельзя зашифровать приватный ключ паролем. Нужна стратегия: либо PIN-код для шифрования ключа, либо хранение ключа только в `sessionStorage` (менее безопасно).

---

### ⬜ Запланировано: Passkey / WebAuthn (Фаза 3)

Биометрический вход (Face ID, Touch ID, Windows Hello). PocketBase пока не поддерживает WebAuthn нативно — потребуется кастомная реализация через `pb_hooks` и библиотеку `@simplewebauthn/server`.

**Приоритет:** Низкий. Реализуем после OAuth2 и стабилизации E2E.

---

### ⬜ Запланировано: Invite-Only режим (опционально)

Для закрытого бета-доступа: коллекция `invite_codes` в PocketBase, валидация через хук `onRecordBeforeCreateRequest`.

## Способы входа

### ✅ Реализовано: Email + Password

`AuthService.loginWithPassword(email, password)` → `pb.collection('users').authWithPassword(...)`.

### ⬜ Будет: OAuth2 (те же провайдеры, что и регистрация)

### ⬜ Будет: Magic Link

PocketBase поддерживает OTP-авторизацию через email начиная с v0.23. Пользователь вводит email → получает код → вводит код → вошёл. Без пароля.

**Когда использовать:** Если пользователь забыл пароль — вместо сброса можно предложить Magic Link.

## Управление сессией

### Как работает сейчас

PocketBase SDK хранит `token` (JWT) и `record` (данные пользователя) в `localStorage` через `LocalAuthStore`. При перезагрузке страницы данные восстанавливаются автоматически.

**Жизненный цикл при старте приложения:**

```
App mount
  → useAuthStore.initialize()
    → AuthService.getLocalRecord()     # Читаем из localStorage (мгновенно)
    → Если есть запись → показываем UI сразу (нет мигания экрана входа)
    → AuthService.refreshSession()     # Фоновая проверка актуальности токена
      → Успех → обновляем pbUser + profile в сторе
      → NetworkError → не разлогиниваем, living в offline-режиме
      → AuthError (токен протух) → разлогиниваем, редирект на /login
```

### Silent Token Refresh (планируется)

PocketBase JWT по умолчанию живёт **7 дней** (настраивается в Settings → Application). При активном использовании приложения нужно продлевать токен до истечения:

```
// Фоновый интервал в auth store
setInterval(async () => {
  if (AuthService.isValid() && document.visibilityState === 'visible') {
    await AuthService.refreshSession();
  }
}, 1000 * 60 * 30); // каждые 30 минут пока открыта вкладка
```

Также продлеваем при `visibilitychange` (пользователь вернулся к вкладке после долгого перерыва).

### Настройка TTL токена в PocketBase

В **PocketBase Admin** → **Settings** → **Application** можно задать:
- `authTokenDuration` — время жизни токена авторизации (по умолчанию 7 дней)
- Для наших нужд рекомендуем **30 дней** — пользователь входит раз в месяц максимум

### Ghost Mode + Сессия

Ghost Mode (PIN-код) — это отдельный слой поверх обычной сессии. Сессия PocketBase при этом **не прерывается**. Ghost Mode только скрывает чаты от постороннего взгляда через UI.

## Защита от ботов

Подход: многоуровневый без CAPTCHA (раздражает пользователей).

| Уровень | Механизм | Где реализован |
|---|---|---|
| **1. Honeypot** | Скрытое поле `_hp`, бот заполняет, сервер отклоняет | Форма + PocketBase hook |
| **2. Time-check** | Форма отправлена < 2 сек после монтирования = бот | Клиент (`useLoginForm`) |
| **3. Rate Limiting** | `useRateLimiter` — 5 попыток, блокировка 60 сек | Клиент (уже реализовано) |
| **4. Server Rate Limit** | PocketBase встроенный + Nginx | Инфраструктура |
| **5. Email верификация** | Отсеивает большинство ботов | После регистрации |
| **Фаза 2** | Cloudflare Turnstile — если атаки усилятся | По необходимости |

## Email верификация

**Принцип: не блокируем вход, но поощряем верификацию.**

```
Регистрация
  → Пользователь сразу попадает в приложение
  → PocketBase отправляет письмо верификации (через Brevo SMTP)
  → Показывается ненавязчивый баннер "Подтвердите email"
    → Закрывается крестиком (флаг в localStorage)
    → Кнопка "Отправить повторно" с кулдауном 60 сек
  → Пользователь переходит по ссылке из письма
    → Открывается /auth/verify?token=...
    → AuthService.confirmVerification(token)
    → Редирект в чат + значок ✅ verified в профиле
```

**Настройка SMTP в PocketBase (Brevo):**

```
PocketBase Admin → Settings → Mail settings:
  SMTP Server:  smtp-relay.brevo.com
  Port:         587
  Username:     [ваш Brevo email]
  Password:     [SMTP Master Key]
  Sender:       noreply@knok-knok.ru
```

**Что даёт верификация:**
- Гарантия реального email
- Доступ к модерации публичных чатов (в будущем)
- Возможность восстановления пароля
