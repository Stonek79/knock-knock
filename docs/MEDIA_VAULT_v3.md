# Media Vault v3: Zero-Knowledge & Offline-First

## Описание
Система Media Vault v3 обеспечивает безопасное хранение, кеширование и отображение медиафайлов (фото, видео, аудио, документы) с использованием архитектуры Zero-Knowledge. Все файлы шифруются на клиенте перед загрузкой на сервер и расшифровываются только на устройстве пользователя.

## Ключевые возможности
1. **Zero-Knowledge Encryption**: Использование AES-GCM (Web Crypto API) для шифрования файлов. Ключи никогда не передаются на сервер в открытом виде.
2. **Offline-First (Dexie.js)**: Автоматическое кеширование расшифрованных блобов в IndexedDB. Это обеспечивает мгновенный доступ к медиа в офлайн-режиме и экономию трафика.
3. **Web Worker Orchestration**: Все тяжелые операции (шифрование, расшифровка, генерация превью) вынесены в `media.worker.ts`, что предотвращает блокировку основного потока UI.
4. **Smart Thumbnails**: Автоматическая генерация и шифрование превью (WebP) для изображений и видео. Превью загружаются первыми, обеспечивая "Wow-эффект" и быструю отрисовку чата.
5. **Vault Mode (Сейф)**: Поддержка перешифровки файлов мастер-ключом пользователя для долгосрочного безопасного хранения в личном Сейфе.

## Технологический стек
- **Web Crypto API**: AES-GCM, PBKDF2.
- **Dexie.js**: Хранилище кеша расшифрованных файлов.
- **Offscreen Canvas**: Генерация превью в воркере.
- **PocketBase**: Хранилище зашифрованных файлов.
- **Zod**: Строгая валидация контрактов между UI и воркером.

## Структура файлов
- `app/src/lib/services/media.ts`: Главный сервис оркестрации (загрузка, скачивание, кеширование).
- `app/src/lib/workers/media.worker.ts`: Логика обработки медиа (шифрование, превью).
- `app/src/lib/db/media-db.ts`: Определение схемы Dexie.
- `app/src/lib/db/useMedia.ts`: Реакт-хук для прозрачного доступа к медиа.
- `app/src/features/chat/message/components/MessageBubble/components/AttachmentRenderer/`: Компоненты отрисовки вложений.

## Поток данных (Upload)
1. Пользователь выбирает файл.
2. `useSendMessage` вызывает `mediaService.uploadMedia`.
3. Воркер генерирует превью и шифрует и оригинал, и превью.
4. Зашифрованные файлы загружаются в PocketBase.
5. Данные (ID, имя, размер, URL превью) сохраняются в JSON вложения сообщения.

## Поток данных (Download/View)
1. `useMedia(mediaId)` проверяет наличие расшифрованного файла в Dexie.
2. Если файла нет, вызывает `mediaService.downloadAndCache`.
3. Файл скачивается из PocketBase, расшифровывается в воркере и сохраняется в Dexie.
4. Хук возвращает `Blob URL` для отображения в `<img>` или `<video>`.
