# AI Агенты и Навыки (Skills)

Этот проект использует систему AI-агентов для автоматизации разработки и поддержки кодовой базы.

## Структура системы агентов
Все конфигурации и навыки агентов находятся в папке `.agent/`.

- `.agent/skills/` — Директория с дополнительными навыками (Skills).
- `.agent/workflows/` — Описания сложных рабочих процессов.

## 🚨 CRITICAL CONSTRAINTS (IMPORTANT)

Эти ограничения среды ОБЯЗАТЕЛЬНЫ для выполнения всеми ИИ-агентами. Нарушение ведет к неработоспособности инструментов.

1.  **NO SEARCHING (Searching is broken)**:
    - Инструмент `searching` (grep_search, search_web и др.) НЕ РАБОТАЕТ.
    - **Вместо этого**: Используй `view_file` для ручного чтения структуры и файлов.
    
2.  **NO PARTIAL EDITING (Editing is broken)**:
    - Инструмент `replace_file_content` / `multi_replace_file_content` НЕ РАБОТАЕТ.
    - **Вместо этого**: Всегда используй `write_to_file` с флагом `Overwrite: true` для обновления файлов. Переписывай файл ЦЕЛИКОМ.

3.  **NO NPX/NODE TOOLS (Environment is limited)**:
    - Инструменты `npm`, `npx`, `biome` могут быть недоступны через `run_command`.
    - **Вместо этого**: Проводи финальный аудит кода и форматирования вручную.

## 🏗️ Архитектурные правила

### 1. PocketBase (pb_hooks)
- **Двойной импорт (Double Require)**: PocketBase v0.23 изолирует контексты. `require` нужен и на верхнем уровне, и внутри обработчиков.
- **Никаких "магических строк"**: Все имена таблиц и полей — в `db.js`.

### 2. Service Worker (VitePWA)
- **self.__WB_MANIFEST**: Должен встречаться в файле `sw.ts` строго **ОДИН** раз для работы плагина `injectManifest`.
- **Типизация**: При нехватке глобальных типов (например, `NotificationAction`) использовать локальные интерфейсы-обертки.

### 3. Чистый код
- **Single line returns**: Однострочные `if (...) return ...` запрещены. Всегда используй `{}` и перенос строки.
- **Документация**: Комментарии, JSDoc и проектная документация — на русском языке.
