# Системная рассылка (Broadcast)

## Описание
Модуль системной рассылки предназначен для отправки глобальных уведомлений всем пользователям платформы Knock-Knock. Оповещения появляются у пользователей в их системном канале (Ghost-чат) или в виде других глобальных нотификаций. Модуль доступен только для пользователей с ролью `admin`.

## Структура
Модуль реализован в соответствии с архитектурой приложения и разделен на три слоя:
1. **UI Layer**: Компонент `BroadcastSettings` (`app/src/features/settings/BroadcastSettings`), который отвечает за отображение формы ввода рассылки и статуса отправки.
2. **Service Layer**: Сервис `broadcastService` (`app/src/lib/services/broadcast.ts`), который содержит бизнес-логику рассылок и выступает связующим звеном.
3. **Repository Layer**: Репозиторий `broadcastRepository` (`app/src/lib/repositories/broadcast.repository.ts`), который осуществляет непосредственные вызовы API (к PocketBase) для создания рассылки.

## Типизация и обработка ошибок
Вызов API использует стандартную для проекта типизацию с обертками `Result` (пакет `neverthrow`) и `AppError`:
- Метод отправки возвращает тип `Promise<Result<void, BroadcastRepoError>>`.
- При возникновении ошибки создается `AppError` с кодом `ERROR_CODES.NETWORK_ERROR` (или другим релевантным).
- В UI-компоненте используется хелпер `getErrorMessage` для безопасного извлечения текста ошибки.

## Использование
Чтобы добавить новый функционал рассылок (например, отложенная рассылка или рассылка по группам):
1. Добавьте метод API в `broadcast.repository.ts`.
2. Опишите бизнес-логику в `broadcast.ts`.
3. Подключите метод в React-компоненте и обработайте `Result`.

**Безопасность**: Защита эндпоинта рассылки (`API_ROUTES.BROADCAST_SEND`) осуществляется на стороне backend, проверяя права администратора (role === 'admin').
