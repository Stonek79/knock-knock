# Руководство по установке PocketBase (Home Server)

Это руководство описывает процесс развертывания PocketBase на домашнем сервере с использованием Docker Compose.

## 0. Быстрая автоматизированная установка (Рекомендуется)

Если у вас настроен SSH-доступ к серверу по ключам, вы можете развернуть всё одной командой прямо с вашего Mac:

```bash
# Из корня проекта
./scripts/pb-setup-server.sh admin alex
```
*(Где `admin` — адрес сервера, `alex` — имя пользователя)*.

**Что сделает этот скрипт:**
1. Создаст папку `~/knok-knok-bd` на сервере.
2. Скопирует `docker-compose.pb.yml` и схему.
3. Запустит PocketBase.
4. Создаст администратора (используя данные из вашего `app/.env`).
5. **Автоматически импортирует все коллекции** (вам не нужно заходить в настройки).

---

## 1. Требования
- Установленный **Docker** и **Docker Compose**.
- Доступ к терминалу сервера.

## 2. Подготовка файлов
Скопируйте содержимое папки `infra/home/` на ваш сервер.

Рекомендуемая структура на сервере:
```
~/knok-knok-bd/
  ├── docker-compose.dev.yml
  ├── pb_data/             # Данные БД
  ├── pb_hooks/            # СЕРВЕРНЫЕ ХУКИ (main.pb.js)
  ├── pb_schema.json       # Схема коллекций
  └── .env                 # Переменные окружения
```

> [!IMPORTANT]
> Папка `pb_hooks` критически важна! Она содержит бизнес-логику для автоматического создания комнаты "Избранное" и каскадного удаления данных.

## 3. Настройка окружения
Создайте файл `.env` в папке с проектом:

```env
PB_ENCRYPTION_KEY_DEV=your_secret_key
```

## 4. Запуск сервера
Для корректной работы хуков и путей в контейнере используется явная команда запуска.

```bash
docker compose -f docker-compose.dev.yml up -d
```

Проверьте логи на наличие загруженных хуков:
```bash
docker logs knok-knok-pb-dev
# Должна быть строка: Loaded 1 hook(s)
```

## 5. Начальная настройка (Admin UI)
1. Откройте в браузере: `https://dev-api.knok-knok.ru:8443/_/` (или ваш локальный IP:8090).
2. Создайте первого администратора.

## 6. Миграция схемы (Collections)
1. Перейдите в **Settings** -> **Import Collections**.
2. Загрузите `pb_schema.json`.
3. Нажмите **Review and Import**.

## 7. Особенности архитектуры
- **Автоматические комнаты**: При регистрации нового пользователя серверный хук автоматически создает для него комнату "Избранное" (Saved Messages). Фронтенду не нужно заботиться о проверке или создании этой комнаты.
- **Каскадное удаление**: При удалении пользователя сервер автоматически удаляет все его сообщения, медиа-файлы и связанные личные комнаты.
