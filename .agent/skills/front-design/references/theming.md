# Темизация и Цвета в Knock-Knock

## Цветовая палитра
Проект использует **Radix UI Themes** в качестве базовой дизайн-системы. Все цвета привязаны к токенам Radix и маппятся на семантические переменные в `index.css`.

### Семантические переменные
Всегда используйте семантические переменные вместо конкретных цветов. Это гарантирует правильную работу Dark/Light режимов.

| Переменная | Назначение |
|------------|------------|
| `--main-background` | Основной фон всего приложения |
| `--surface` | Фон панелей, модальных окон, карточек |
| `--foreground` | Основной цвет текста |
| `--muted-foreground` | Цвет второстепенного текста (метаданные, даты) |
| `--primary` | Акцентный цвет (кнопки, активные элементы) |
| `--border` | Цвет разделителей и границ элементов |

### Специальные переменные мессенджера
Для чатов используются специфичные токены:
- `--bubble-out`: Фон исходящего сообщения.
- `--bubble-out-fg`: Текст исходящего сообщения.
- `--bubble-in`: Фон входящего сообщения.
- `--bubble-in-fg`: Текст входящего сообщения.
- `--bubble-meta`: Цвет времени и статуса сообщения.

## Как работать с темами в коде

### 1. Radix Themes `color` проп
Radix автоматически подставляет нужный оттенок из палитры в зависимости от темы:
```tsx
<Text color="blue">Этот текст будет синим в обеих темах</Text>
<Button color="accent">Акцентная кнопка</Button>
```

### 2. Использование переменных в CSS Modules
Для кастомных компонентов используйте переменные из `.radix-themes` области `index.css`:
```css
.customPanel {
    background-color: var(--surface);
    color: var(--foreground);
    border: 1px solid var(--border);
}
```

### 3. Проверка темной темы
При разработке UI через `front-design` всегда проси агента проверить:
- Читаемость текста на `--surface` в темной теме.
- Контрастность `--muted-foreground`.
- Яркость акцентных цветов (они не должны "резать глаз" в темноте).

## Принципы премиального дизайна в темах
1. **Glassmorphism**: Используйте `backdrop-filter: blur(...)` вместе с полупрозрачным `--surface` для верхних панелей.
2. **Subtle Shadows**: В темной теме тени должны быть более глубокими, но менее насыщенными по цвету.
3. **Consistency**: Не используйте цвета, которых нет в `index.css` или палитре Radix.
