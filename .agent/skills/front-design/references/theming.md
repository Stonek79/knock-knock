# Темизация и Цвета

В Knock-Knock используется Headless UI подход. Все цвета и темы управляются исключительно через CSS-переменные, определенные в глобальном `index.css`. Мы **не используем** CSS Layers (`@layer`), чтобы избежать конфликтов специфичности со сторонними библиотеками.
## 🎨 Система токенов

Все переменные живут глобально в `index.css`, что гарантирует их доступность для всех CSS-модулей в проекте.
| Группа | Переменная | Назначение |
|------------|------------|------------|
| App | `--bg-app` | Общий фон приложения |
| Surface | `--surface` | Основной фон панелей |
| Surface | `--surface-1` | Вложенный (secondary) фон |
| Surface | `--surface-2` | Фон заголовков и тулбаров |
| Text | `--foreground` | Основной текст |
| Text | `--muted` | Вторичный текст |
| Accents | `--accent-primary` | Главный акцент (зеленый/циан и тд) |
| Layout | `--max-w-settings` | 1200px (лимит для страниц настроек) |

## 📦 Локальные токены компонентов

**Правило:** Не используй глобальный токен напрямую в стилях CSS-модуля. Пробрасывай его через локальную переменную.

### Плохо:
```css
.card { background: var(--surface); }
```

### Хорошо:
```css
.card {
  --card-bg: var(--surface);
  background: var(--card-bg);
}
```
Это позволяет менять стиль конкретной карточки в конкретном месте без правок всего компонента.

## ✨ Работа с Container Queries

Для правильной темизации адаптивных блоков используй `@container`. Это позволяет компоненту менять цветовую схему в зависимости от его местоположения (например, в Sidebar или Chat).

## 🌑 Темная тема

При проверке темной темы через `agent` убедись, что:
1. Используется правильный уровень свечения (`--title-glow`) для Неона/Эмеральда.
2. Контрастность текста соответствует WCAG AA.
3. Shadows не пропадают (в темной теме они должны быть более глубокими).
