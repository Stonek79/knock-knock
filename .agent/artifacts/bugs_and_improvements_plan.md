## 🔴 ГРУППА 1 — Критические баги (регрессии)

### Баг 1.1 — Медиа не загружается (404 на temp-id)

**Симптом:** `useMedia [temp-file-UUID-имяфайла]: ошибка загрузки` → 404
**Причина:** В PocketBase логах нет ни одного POST к `/api/collections/media/records/` — значит `mediaService.uploadMedia` падает до сетевого запроса. Воркер получает `cryptoKey = undefined` (консоль: `⚠️ DEV-FALLBACK: Ключ комнаты не найден`). `onSettled` делает `invalidateQueries`, React Query рефетчит по temp-id — 404.
**Файлы:** `useSendMessage.ts`, `media.worker.ts`, `useRoomKey` / криптографический провайдер

**Диагностика:** Нужно проверить как roomKey попадает в useSendMessage и почему он `undefined`.

---

### Баг 1.2 — Speech Recognition падает сразу

**Симптом:** `Speech recognition error {isTrusted: true}` сразу при старте, ещё до любого звука.
**Причина:** В `onerror` обработчике не читается `event.error` — браузер возвращает объект с `error: "not-allowed"` или `"service-not-allowed"` (проблема permissions/HTTPS). В dev-окружении Chrome может блокировать микрофон без HTTPS.
**Файл:** `useSpeechRecognition.ts`

---

### Баг 1.3 — Сортировка чатов сломана

**Симптом:** Новые сообщения не поднимают чат вверх.
**Причина:** `getRoomsWithMembers` не передаёт `sort` в PocketBase запрос → порядок случайный. Клиентская сортировка по `_rawDate` правильная, но `last_message.created_at` может быть `null` (у комнат без сообщений), и все они получают `DEFAULT_DATE` — одинаковое значение.
**Файл:** `room.repository.ts` → `getRoomsWithMembers`

## 🟡 ГРУППА 2 — Функциональные баги

### Баг 2.1 — Скролл обрезает картинку

**Симптом:** Скролл к последнему сообщению — виден только верхний край изображения.
**Причина:** `scrollIntoView` вызывается до того, как `<img>` загрузилась и получила высоту. Контейнер не знает реальных размеров.
**Файл:** `MessageList` / хук управления скроллом

### Баг 2.2 — Неактивные кнопки Zoom в галерее
**Симптом:** При открытии фото в Lightbox кнопки "+" и "-" отображаются, но недоступны для нажатия.
**Причина:** Плагину Zoom, вероятно, не хватает данных о реальных размерах изображения (width/height в массиве slides) или ограничений maxZoomPixelRatio.
**Файл:** `useLightboxSlides.ts` / `MessageBubble/index.tsx`

### Баг 2.3 — Логика выделения сообщений (как в Telegram)
**Симптом:** Простой левый клик по сообщению на десктопе выделяет его, что ломает UX.
**Причина:** Обработчик `onClick` всегда вызывает `onToggleSelection`.
**Решение:** Клик должен выделять сообщение *только* если уже активно "режим выделения" (выбрано > 0 сообщений).
**Файл:** `MessageBubble/index.tsx` (потребуется пропс `isSelectionMode`)
