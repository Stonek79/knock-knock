# Эпик: Рефакторинг и реализация Reply (Цитаты) и Forward (Пересылка)
**СТАТУС: ✅ ПОЛНОСТЬЮ ЗАВЕРШЕНО**

## 1. Схема БД и Zod (Строгая типизация)

Поскольку PocketBase использует JSON-поле `metadata` для хранения дополнительных атрибутов сообщения, **нам не нужно менять структуру таблиц в самой БД**. База данных уже готова к приему этих данных. Нам нужно обновить только Zod-схемы на клиенте.

**Что нужно для Reply (Цитата):**
- `reply_to_id: string` (ID оригинального сообщения).
*(Мы уже добавили это в схему, рендеринг работает).*

**Что нужно для Forward (Пересылка):**
В отличие от цитаты, пересланное сообщение часто теряет связь с оригинальным чатом (если переслали из другой комнаты), поэтому нам нужно сохранить имя оригинального автора текстом.
- `forward_from_name?: string` (Имя оригинального автора).
- `forward_from_id?: string` (Опционально, ID пользователя).

### План действий по схемам (`schemas/message.ts`):
1. Добавить `reply_to_id: z.string().optional()`.
2. Добавить `forward_from_name: z.string().optional()`.
3. Добавить `forward_from_id: z.string().optional()`.
