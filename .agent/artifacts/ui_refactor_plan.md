# План UI-улучшений (Полировка интерфейса)

> **Статус:** 🟢 В процессе
> **Приоритет:** Улучшение пользовательского опыта (UX)

Этот план включает задачи по визуальному улучшению отображения сообщений и вложений в чате, чтобы приложение ощущалось как премиальный мессенджер (в стиле Telegram / WhatsApp).

---

## 1. Карточки документов (File/Document Attachments)
**Проблема:** Сейчас не-медиа файлы (PDF, DOCX, ZIP) отображаются неприглядно или как обычные ссылки.
**Решение:**
- Создать компонент `DocumentAttachmentCard`.
- Визуально: иконка документа (цвет зависит от расширения, например, красный для PDF, синий для Word), название файла с обрезкой по центру (`file...name.pdf`), размер файла (форматированный в KB/MB).
- Кнопка загрузки со спиннером во время скачивания и расшифровки.

## 2. Фото-галерея вложений (Media Grid / Mosaic)
**Проблема:** Если отправить 4 фотографии, они выстраиваются длинной вертикальной "простыней", занимая весь экран.
**Решение:**
- Создать компонент `MediaGallery`.
- Реализовать алгоритм "плитки" (Masonry/Grid):
  - 1 фото: на всю ширину пузыря.
  - 2 фото: делят пространство 50/50 по вертикали.
  - 3 фото: 1 большое сверху, 2 маленьких снизу.
  - 4+ фото: сетка 2x2. Если фото больше 4, на последнем превью рисуется полупрозрачный слой с текстом `+N`.

## 3. Редизайн блока цитирования (Reply Block)
**Проблема:** Цитируемые сообщения внутри пузыря могут выглядеть скучно или сливаться с основным текстом.
**Решение:**
- Добавить красивую левую границу (border-left) цвета `accent-primary` для цитат.
- Имя автора цитаты должно быть акцентным цветом.
- Если цитируется медиа или файл — добавлять маленькую иконку (📸 Фото, 🎤 Голосовое) перед текстом цитаты.

## 4. Превью ссылок (OpenGraph Link Previews)
**Проблема:** Отправленные ссылки на сайты выглядят просто как синий текст.
**Решение:**
- Написать утилиту, которая парсит текст сообщения на наличие URL.
- Если URL найден — отображать красивую карточку под сообщением (картинка сайта, заголовок, краткое описание).
- *(Требует написания небольшого парсера на стороне сервера PocketBase, чтобы обходить CORS).*

## 5. Микроанимации (Microinteractions)
**Проблема:** Элементы появляются резко.
**Решение:**
- Плавное появление (Fade-in + Slide-up) для новых сообщений в чате.
- Анимация вызова контекстного меню (шторки реакций).
- Скелетоны для загружающихся медиа.
