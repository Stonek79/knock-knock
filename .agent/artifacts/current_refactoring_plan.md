## Этап 5: Исправление кэширования зашифрованных Blob (КРИТИЧЕСКИЙ БАГ) 🔴
**Цель:** Исправить баг, при котором отправленные картинки не отображаются без хард-релоада.

**Корневая причина:** Метод `uploadMedia` кэширует в IndexedDB **зашифрованный** Blob (результат воркера после `COMPRESS_IMAGE` / `ENCRYPT_BLOB`). При последующем чтении `ensureThumbnail` → `getMedia` находит запись в кэше, возвращает зашифрованный Blob → `URL.createObjectURL(зашифрованный_blob)` → **битая картинка**. После хард-релоада кэш очищается и `ensureThumbnail` скачивает файл с сервера → расшифровывает → всё работает.

1. [ ] **`media.service.ts` → `uploadMedia` (строки 193-216)**
   - Заменить кэшируемые данные с `original` (зашифрованного Blob из воркера) на исходный незашифрованный `file`:
     ```
     [MEDIA_CACHE_FIELDS.BLOB]: file instanceof Blob ? file : null,
     ```
   - Для `thumbnail`: воркер при `COMPRESS_IMAGE` генерирует сжатое превью **до** шифрования — нужно проверить, доступно ли оно. Если нет — кэшировать `null`, пусть `ensureThumbnail` скачает потом.
   - Альтернатива: если воркер не возвращает незашифрованные промежуточные данные, **полностью убрать кэширование из `uploadMedia`** (удалить блок строк 186-220) и позволить `ensureThumbnail` отработать штатно при первом рендере.

2. [ ] **Проверить контракт `mediaWorkerClient.postTask`**
   - Посмотреть, возвращает ли воркер и незашифрованные, и зашифрованные варианты.
   - Если возвращает только зашифрованный — реализовать альтернативу (удалить кэш из upload).

## Этап 6: Удаление сообщений "Без следов" (Hard Delete) 🔴
**Цель:** Привести логику удаления в соответствие с концепцией "Удалил и забыл" (как в Telegram), обеспечить консистентность кэша и избежать 404 ошибок на старых артефактах.

### Бизнес-правила удаления:
1. **Своё сообщение (или Админ)**: Физическое удаление (Hard Delete) из БД. Сообщение мгновенно исчезает у всех участников.
   - Реакции (Эмодзи) удаляются автоматически базой данных (настроен Cascade Delete в PocketBase).
   - Прикрепленные медиафайлы (`attachments`) предварительно физически удаляются из облака и из локального кэша `Dexie`.
2. **Чужое сообщение (у себя)**: Локальное скрытие (`Local Hide`). Добавляем свой `userId` в массив `metadata.deleted_by`. Сообщение исчезает только для нас. Медиа удаляется из локального кэша `Dexie`, но остается в облаке для остальных.
3. **Очистка комнаты (`clearRoom`)**: Локальное скрытие всех сообщений комнаты через добавление `userId` в `metadata.deleted_by`. Медиа комнаты полностью удаляется из локального кэша `Dexie`.

### План исправлений:

1. [ ] **Уровень Репозитория (`message.repository.ts`)**
   - Добавить метод `hardDeleteMessage(messageId: string)` для физического удаления.
   - Переписать `clearRoom` на использование локального скрытия: добавление `userId` в массив `metadata.deleted_by` с безопасной проверкой типов `typeof r.metadata === "object"`, оператором `in` и фильтрацией массива (без кастований `as`).

2. [ ] **Уровень Сервиса (`message.ts`)**
   - Обновить логику `deleteMessage`:
     - Для глобального удаления: удалять файлы из облака (`mediaService.deleteMedia`), затем вызывать `messageRepository.hardDeleteMessage`.
     - Для локального скрытия: безопасно (через `?.`) обновлять `deleted_by`.
   - Добавить "предохранители": если сообщение уже удалено глобально (`is_deleted`) или скрыто локально, сразу возвращать успех (`ok()`), чтобы избежать 404 ошибки от PocketBase.

3. [ ] **Уровень UI и Синхронизации**
   - **Realtime**: Ничего менять не нужно, `chat-realtime.ts` уже умеет обрабатывать `REALTIME_ACTIONS.DELETE` и вырезать сообщение из React Query.
   - **`useMessages.ts`**: Добавить безопасную проверку `deleted_by` через Optional Chaining `?.` и `Array.isArray`, чтобы избежать падений на старых данных.
   - **`MessageBubble` & `useMessageSelection.ts`**: Запретить клики, выделения и лонг-прессы по старым артефактным сообщениям (`is_deleted: true`), чтобы не давать пользователю пытаться удалить их повторно.
   - **Цитаты (ReplyBlock)**: Добавить обработку случая, когда цитируемое сообщение физически удалено из кэша (выводить плашку "Сообщение удалено" внутри самой цитаты).
4. [ ] **Локализация (`useChatActions.ts`)**
   - Перевести сообщения об ошибках удаления и редактирования на `i18n`: `t("chat.deleteFailed", "Не удалось удалить сообщение")`.
