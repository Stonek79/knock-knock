Always use Context7 MCP when I need library/API documentation, code generation, setup or configuration steps without me having to explicitly ask.
Используй все имеющиеся возможности MCP.
Всегда взаимодействуй со мной на русском языке.
Всегда используй правила проекта при генерации кода.
Прежде чем что-то сделать - составь план и обдумай его, предложи 3 варианта решения, обоснуй каждый с описанием плюсов и мнусов, предожи свой вариант и обоснуй выбор. Спроси меня есть ли замечани и предожения к предложенному тобой плану. Приступай к реализации только после моего одобрения без вопросов и предожений с моей стороны.
Всегда проводи линтинг и тестирование после выполнения. Задача не закончена, если есть. ошибки.
При использовании типизации избегаем any и кастования типов, используем всю мощь последних версий typescript.
При расзработке архитектуры используем чистый код и грамотное модульное структурироваие файловой системы проекта, выделяем блоки по сущностям и логческим связям.
Код должен быть задокументирован. Где можно и нужно - используем jsdoc, в других необходимых местах просто оставляем комментарии с описанием. Документируем на русском. Не забываем делать общую документацию по проекту с описанием его целей, используемых инструментов и подходов, структуры, механизмов запуска и использования, для чего в корне проекта должна быть отдельная папка.
UI/UX: Верстка с использованием красивой и удобной библиотеки UI подходящей под конкретный случай, адаптация под мобильные экраны (Mobile First). UI подход выбирается в начале старта проекта и должен использоваться на протяжении всей разработки, если в процессе не будет изменен. В случае изменения в процессе разработки необходимо использовать согласованный подход к реализации UI. В проекте используем Radix Theme и Radix UI, в сочетании с css модулями, инлайн стили (styles) используем только когда другой подход не оправдан.
Всегда используй последние версии библиотеки, если в проекте специально не требуется использовать конкретные версии. Анализируй код на наличие старых и менее эффективных подходов, методов и функций и предлагай замену либо используй при кодинге более современные и эффективные.
При работе в режиме агента используй браузер для тестирования корректной работы UI.
